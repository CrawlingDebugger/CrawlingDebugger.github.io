Haiku Generator Task Write-Up

3a: Program Purpose and Function

Purpose: The program aims to automate creative writing, allowing users to rapidly generate and construct three-line structured Japanese haiku poems. Function: The user clicks a trigger button, which processes separate arrays populated with structured syllables (5-7-5 configuration), randomizing text outputs to update structural UI labels.

3b: Data Abstraction

List Implementation: Three dynamic arrays named fiveSyllableList1, sevenSyllableList, and fiveSyllableList2 hold structural strings representing noun-verb descriptors, actions, and closing atmospheric observations. Managing Complexity: Storing these values inside arrays allows the generation function to traverse an unlimited amount of structural variations through unified function code blocks rather than declaring individual, hard-coded scalar string variables for every single poem variation.

3c: Managing Complexity & Selection

Without lists, hundreds of global text lines would need separate conditional evaluation branches, creating spaghetti code that is nearly impossible to debug or scale as new poetry terms are introduced.

Reminders App Task Write-Up

3a: Program Purpose and Function

Purpose: Helps students and developers manage tasks to prevent cognitive overload during busy project-building sprints. Function: Users can input tasks, schedule time periods, view tasks dynamically using sequential forward/backward index navigators, and clear them once completed.

3b: Data Abstraction

List Implementation: An array named taskStorageList collects user inputs. Managing Complexity: The array preserves task entries across screen transitions, preventing UI states from refreshing into blank views. Index pointers prevent out-of-bounds selection errors on navigation buttons.

3c: Algorithm & Control Structures

Uses a tracking index pointer currentIndex which decrements or increments during list traversals. Conditional statement blocks verify if indices lie within boundaries (currentIndex < taskStorageList.length) before performing screen transitions.

Like & Commenting App Task Write-Up

3a: Program Purpose and Function

Purpose: Simulates a social interaction feed where users express feedback (likes) and add text reactions to persistent dynamic feeds. Function: Event listeners respond to click actions, calling helper modules to update variable values on screen instantly.

3b: Dynamic Redrawing Pattern

Flow: Action taken → State variable changes → Update database state → Run redraw UI module. This clean structural breakdown represents abstraction in production environments.

3c: Managing Complexity

Encapsulating all interface redrawing procedures into an individual named procedure, updateFeedView(), avoids duplicating redundant UI text setters across every click listener, proving deep abstract planning.

Core Programming Concepts

These foundational programming concepts were applied to formulate code structures across the three digital projects.

Lists manage dynamic sizes of structured data by organizing values into zero-indexed slots. This streamlines memory allocation, letting index routines dynamically grab terms like poem fragments or calendar items.

Example: let syllables = ["ocean wave", "autumn mist", "glowing sky"];

Control structures dictate code execution paths. Selection blocks (if-else checks) direct variables according to comparison results. Iteration paths (for loops, while loops) evaluate lines repeatedly until state conditions switch.

Subdividing complex applications into small, reusable functions simplifies debugging. Creating dynamic modules ensures codebase flexibility, keeping software systems functional even under heavy logic load additions.
